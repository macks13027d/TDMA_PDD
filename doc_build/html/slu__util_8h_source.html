115 /*
116  *-- This contains the options used to control the solve process.
117  *
118  * Fact (fact_t)
119  * Specifies whether or not the factored form of the matrix
120  * A is supplied on entry, and if not, how the matrix A should
121  * be factorizaed.
122  * = DOFACT: The matrix A will be factorized from scratch, and the
123  * factors will be stored in L and U.
124  * = SamePattern: The matrix A will be factorized assuming
125  * that a factorization of a matrix with the same sparsity
126  * pattern was performed prior to this one. Therefore, this
127  * factorization will reuse column permutation vector
128  * ScalePermstruct->perm_c and the column elimination tree
129  * LUstruct->etree.
130  * = SamePattern_SameRowPerm: The matrix A will be factorized
131  * assuming that a factorization of a matrix with the same
132  * sparsity pattern and similar numerical values was performed
133  * prior to this one. Therefore, this factorization will reuse
134  * both row and column scaling factors R and C, both row and
135  * column permutation vectors perm_r and perm_c, and the
136  * data structure set up from the previous symbolic factorization.
137  * = FACTORED: On entry, L, U, perm_r and perm_c contain the
138  * factored form of A. If DiagScale is not NOEQUIL, the matrix
139  * A has been equilibrated with scaling factors R and C.
140  *
141  * Equil (yes_no_t)
142  * Specifies whether to equilibrate the system (scale A's row and
143  * columns to have unit norm).
144  *
145  * ColPerm (colperm_t)
146  * Specifies what type of column permutation to use to reduce fill.
147  * = NATURAL: use the natural ordering
148  * = MMD_ATA: use minimum degree ordering on structure of A'*A
149  * = MMD_AT_PLUS_A: use minimum degree ordering on structure of A'+A
150  * = COLAMD: use approximate minimum degree column ordering
151  * = MY_PERMC: use the ordering specified in ScalePermstruct->perm_c[]
152  *
153  * Trans (trans_t)
154  * Specifies the form of the system of equations:
155  * = NOTRANS: A * X = B (No transpose)
156  * = TRANS: A**T * X = B (Transpose)
157  * = CONJ: A**H * X = B (Transpose)
158  *
159  * IterRefine (IterRefine_t)
160  * Specifies whether to perform iterative refinement.
161  * = NO: no iterative refinement
162  * = WorkingPrec: perform iterative refinement in working precision
163  * = ExtraPrec: perform iterative refinement in extra precision
164  *
165  * PrintStat (yes_no_t)
166  * Specifies whether to print the solver's statistics.
167  *
168  * DiagPivotThresh (double, in [0.0, 1.0]) (only for sequential SuperLU)
169  * Specifies the threshold used for a diagonal entry to be an
170  * acceptable pivot.
171  *
172  * PivotGrowth (yes_no_t)
173  * Specifies whether to compute the reciprocal pivot growth.
174  *
175  * ConditionNumber (ues_no_t)
176  * Specifies whether to compute the reciprocal condition number.
177  *
178  * RowPerm (rowperm_t) (only for SuperLU_DIST)
179  * Specifies whether to permute rows of the original matrix.
180  * = NO: not to permute the rows
181  * = LargeDiag: make the diagonal large relative to the off-diagonal
182  * = MY_PERMR: use the permutation given in ScalePermstruct->perm_r[]
183  *
184  * ReplaceTinyPivot (yes_no_t) (only for SuperLU_DIST)
185  * Specifies whether to replace the tiny diagonals by
186  * sqrt(epsilon)*||A|| during LU factorization.
187  *
188  * SolveInitialized (yes_no_t) (only for SuperLU_DIST)
189  * Specifies whether the initialization has been performed to the
190  * triangular solve.
191  *
192  * RefineInitialized (yes_no_t) (only for SuperLU_DIST)
193  * Specifies whether the initialization has been performed to the
194  * sparse matrix-vector multiplication routine needed in iterative
195  * refinement.
196  */
